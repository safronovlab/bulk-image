# SPEC: core/zip_builder.py

> **Слой**: Core  
> **Зависимости**: zipfile (stdlib), io (stdlib)  
> **Ответственность**: Создание ZIP-архива из обработанных изображений с правильной структурой папок  
> **Соседний файл**: `zip_builder.py`

---

## 1. НАЗНАЧЕНИЕ

Модуль формирует финальный ZIP-архив с результатами обработки. Поддерживает две структуры: плоскую (один результат на изображение) и вложенную (несколько вариаций одного изображения).

---

## 2. ФУНКЦИИ

### 2.1 `build_zip`

**Главная функция модуля.**

**Вход:**
- `results`: список кортежей, каждый из которых содержит:
  - `original_filename`: строка — оригинальное имя файла без расширения (например `skull_tee`)
  - `variation_name`: строка или None — имя вариации (например `Jordan_Retro_Blue`). None = единственная вариация
  - `png_bytes`: bytes — содержимое PNG-файла (результат обработки)

**Выход:** bytes — содержимое ZIP-файла

**Управление памятью:** ZIP РЕКОМЕНДУЕТСЯ записывать напрямую на диск (tempfile в RESULT_DIR) вместо BytesIO. Передавать в build_zip пути к файлам на диске вместо bytes в RAM. Пиковое потребление при BytesIO: 300MB ZIP + 300MB results list = ~600MB на одну job. При 5 concurrent jobs = 3GB, что может вызвать OOM-kill без graceful shutdown. Если BytesIO используется — вызывающий код ОБЯЗАН освобождать results list перед сборкой ZIP.

**Защита от Zip Slip:** Перед записью каждого файла в ZIP ОБЯЗАТЕЛЬНА проверка: путь внутри ZIP не должен начинаться с `/` и не должен содержать `..`. Это defence-in-depth поверх sanitize_filename — гарантирует невозможность записи в произвольный путь при распаковке клиентом.

**Алгоритм:**
1. Создать `BytesIO` буфер
2. Создать `ZipFile` в режиме записи (`'w'`) с компрессией `ZIP_STORED` (без дополнительного сжатия — PNG уже сжат)
3. **Определить структуру**: проверить, есть ли у какого-либо `original_filename` более одной вариации
4. **Для каждого результата** определить путь внутри ZIP:
   - Если `variation_name` is None: путь = `{original_filename}_recolored.png`
   - Если `variation_name` не None: путь = `{original_filename}/{variation_name}.png`
5. **Проверить уникальность** путей: если два результата дают одинаковый путь — добавить суффикс `_2`, `_3` и т.д.
6. **Записать** каждый PNG в ZIP: `zipfile.writestr(path, png_bytes)`
7. **Закрыть** ZipFile
8. **Вернуть** `buffer.getvalue()`

**Нюансы:**
- `ZIP_STORED` вместо `ZIP_DEFLATED`: PNG уже использует deflate-сжатие, повторное сжатие бесполезно и тратит CPU
- Пути в ZIP используют `/` (forward slash) — стандарт ZIP-формата, работает на всех ОС

---

### 2.2 `sanitize_filename`

**Очистка имени файла для безопасного использования в ZIP.**

**Вход:** строка — оригинальное имя файла

**Выход:** строка — безопасное имя файла

**Алгоритм:**
1. Убрать расширение файла (всё после последней точки)
2. Заменить все символы кроме `[a-zA-Z0-9_-]` на `_`
3. Убрать множественные подчёркивания подряд: `__` → `_`
4. Trim `_` с начала и конца
5. Если результат пустой — вернуть `"unnamed"`
6. Обрезать до 100 символов

---

## 3. ГРАНИЧНЫЕ СЛУЧАИ

- Один результат: ZIP содержит один файл `{name}_recolored.png`
- 20 результатов без вариаций: 20 файлов `{name1}_recolored.png`, `{name2}_recolored.png`, ...
- 20 результатов × 3 вариации: 20 папок по 3 файла
- Дублирующиеся имена файлов: автоматическое добавление суффикса `_2`, `_3`
- Пустой список results: вернуть пустой ZIP (валидный ZIP без записей)
- Имя файла с Unicode/спецсимволами: sanitize_filename очищает

---

## 4. ПРИМЕР СТРУКТУРЫ ZIP

**Без вариаций:**
```
results.zip
├── skull_tee_recolored.png
├── stay_true_tee_recolored.png
└── voodoo_doll_tee_recolored.png
```

**С вариациями:**
```
results.zip
├── skull_tee/
│   ├── Jordan_Retro_Blue.png
│   └── Jordan_Bred_Red.png
├── stay_true_tee/
│   ├── Jordan_Retro_Blue.png
│   └── Jordan_Bred_Red.png
└── voodoo_doll_tee/
    ├── Jordan_Retro_Blue.png
    └── Jordan_Bred_Red.png
```

---

## 5. ТЕСТОВЫЕ СЦЕНАРИИ (для QA)

1. Один файл без вариаций: ZIP содержит `{name}_recolored.png`
2. Три файла с вариациями: ZIP содержит 3 папки по N файлов
3. Дублирующиеся имена: суффиксы добавляются корректно
4. Пустой список: валидный пустой ZIP
5. Имя с кириллицей/спецсимволами: sanitize_filename очищает
6. ZIP открывается стандартными средствами macOS/Windows
