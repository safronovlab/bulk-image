# SPEC: core/color_extractor.py

> **Слой**: Core  
> **Зависимости**: numpy, OpenCV (cv2), scikit-learn (KMeans)  
> **Ответственность**: Извлечение цветовой информации из изображения — eyedropper, dominant colors, suggest-mappings  
> **Соседний файл**: `color_extractor.py`

---

## 1. НАЗНАЧЕНИЕ

Модуль анализа цветов изображения. Три функции:
1. **Eyedropper** — получить цвет конкретного пикселя по координатам
2. **Dominant Colors** — K-Means кластеризация для определения основных цветов
3. **Suggest Mappings** — автоматический подбор маппингов из dominant colors к target palette

---

## 2. ФУНКЦИИ

### 2.1 `pick_color`

**Вход:**
- `image_rgba`: numpy массив (H, W, 4), dtype uint8
- `x`: целое число (координата по горизонтали)
- `y`: целое число (координата по вертикали)

**Выход:** объект ColorInfo (hex, rgb, lab)

**Алгоритм:**
1. Проверить границы: `0 <= x < W` и `0 <= y < H`. Если за пределами — выбросить ValueError
2. Извлечь пиксель: `pixel = image_rgba[y, x]` (numpy: строка=y, столбец=x)
3. Взять R, G, B из первых трёх каналов (alpha игнорируется)
4. Конвертировать RGB → HEX (формат `#RRGGBB`)
5. Конвертировать RGB → LAB (через вспомогательную функцию из color_engine или локальную)
6. Вернуть ColorInfo со всеми тремя представлениями

---

### 2.2 `extract_dominant_colors`

**Вход:**
- `image_rgba`: numpy массив (H, W, 4), dtype uint8
- `count`: целое число 1-20 (количество кластеров, по умолчанию 5)

**Выход:** список объектов DominantColor, отсортированный по percentage (убывание)

**Алгоритм:**
1. **Развернуть** изображение в массив пикселей: (H*W, 4)
2. **Отфильтровать** прозрачные пиксели: оставить только те, где alpha >= 128
3. Если после фильтрации осталось 0 пикселей — вернуть пустой список
4. **Извлечь** RGB-каналы из оставшихся: (N, 3)
5. **Subsample**: если N > 50000 — случайная выборка 50000 пикселей (для скорости K-Means). ОБЯЗАТЕЛЬНО использовать фиксированный seed для генератора случайных чисел (np.random.default_rng(42)) — иначе «детерминированный» K-Means будет получать разные данные на каждом вызове
6. **Конвертировать** RGB → LAB (через OpenCV, batch-конверсия)
**GIL-блокировка:** KMeans вызывает C-extension, удерживающую GIL. Все вызовы extract_dominant_colors ОБЯЗАНЫ выполняться через asyncio.to_thread() или loop.run_in_executor() на уровне вызывающего сервиса — иначе asyncio event loop заблокирован, healthcheck не отвечает, orchestrator перезапустит контейнер.

7. **Запустить KMeans** с K = count, в LAB-пространстве
   - `n_init=10` для стабильности
   - `max_iter=300` (для batch_analyze рекомендуется max_iter=100 или MiniBatchKMeans — при 20 изображениях × count=20 × n_init=10 = до 4000 итераций, что блокирует worker на 40+ секунд)
   - `random_state=42` для воспроизводимости
8. **Для каждого кластера:**
   a. Центроид в LAB → конвертировать обратно в RGB → HEX
   b. Количество пикселей в кластере / общее количество * 100 → percentage
9. **Сортировать** по percentage (убывание)
10. **Вернуть** список DominantColor

**Нюансы:**
- K-Means в LAB-пространстве даёт перцептуально лучшие кластеры, чем в RGB
- Subsample до 50000 — баланс между точностью и скоростью (K-Means на 50k точках ~0.3 сек)
- random_state фиксирован для детерминированности (один и тот же результат при повторном вызове)
- **Производительность при batch_analyze:** При обработке 20 изображений K-Means может блокировать worker на 6-20 секунд. Для batch_analyze ОБЯЗАТЕЛЬНО выполнять обработку через asyncio.to_thread(). Рекомендуется рассмотреть MiniBatchKMeans как drop-in замену (в 3-5x быстрее)

### 2.3 `suggest_mappings`

**Вход:**
- `image_rgba`: numpy массив (H, W, 4), dtype uint8
- `target_palette`: список строк HEX (1-10 цветов)

**Выход:** список объектов MappingSuggestion, отсортированный по from_percentage (убывание)

**Оптимизация вычислений:** suggest_mappings вызывает extract_dominant_colors (KMeans CPU-bound) + жадный алгоритм. Рекомендуется кэширование результатов extract_dominant_colors по image_id (TTL-кэш или сохранение при первом вызове) — повторный suggest для того же изображения не должен пересчитывать KMeans.

**Алгоритм:**
1. **Извлечь dominant colors** исходного изображения: вызвать `extract_dominant_colors(image_rgba, count=10)`
2. **Конвертировать** все dominant colors в LAB
3. **Конвертировать** все target palette цвета в LAB
4. **Для каждого target-цвета** найти ближайший dominant color по Delta-E (CIE76):
   a. Рассчитать Delta-E между target и КАЖДЫМ dominant color
   b. Выбрать пару с минимальным Delta-E
   c. **Жадный алгоритм без дубликатов**: если dominant color уже назначен другому target — пропустить, взять следующий ближайший
5. **Рассчитать confidence** для каждой пары: `confidence = 1.0 - (delta_e / MAX_DELTA_E)`, где MAX_DELTA_E = 50.0
6. **Отфильтровать** пары с confidence < 0.3 (слишком далёкие — нет смысла предлагать)
7. **Сортировать** по from_percentage (убывание) — сначала крупные области дизайна
8. **Вернуть** список MappingSuggestion

**Логика жадного алгоритма:**
- Сортировать target-цвета по порядку подачи (пользователь задаёт приоритет)
- Для каждого target по очереди: найти ближайший свободный dominant color
- Это гарантирует что каждый dominant color назначается максимум одному target
- Если target-цветов больше чем dominant — оставшиеся targets не получают предложений

## 3. ЗАЩИТА ОТ CPU-DOS

- **Таймаут на batch_analyze:** Вызывающий сервис ОБЯЗАН устанавливать таймаут 30 секунд на весь batch-вызов. При превышении — прерывать с HTTP 408 или 504
- **Рекомендация для batch-режима:** Использовать MiniBatchKMeans как drop-in замену (в 3-5x быстрее). При count=20 и n_init=10 стандартный KMeans запускает 200 итераций на одно изображение — для batch это неприемлемо
- **Интеграционный тест:** Healthcheck ОБЯЗАН отвечать менее чем за 5 секунд во время batch_analyze. Все CPU-bound вызовы (extract_dominant_colors, suggest_mappings) ОБЯЗАНЫ выполняться через asyncio.to_thread() на уровне вызывающего сервиса

---

## 4. ГРАНИЧНЫЕ СЛУЧАИ

- Изображение полностью прозрачное (alpha=0 у всех пикселей): `extract_dominant_colors` → пустой список, `suggest_mappings` → пустой список
- Изображение одного цвета: K-Means вернёт один кластер с percentage ~100%
- target_palette содержит цвет, которого нет в изображении: Delta-E будет высоким → confidence < 0.3 → отфильтруется
- count > количества уникальных цветов: K-Means выдаст меньше кластеров или дубликаты — поведение scikit-learn по умолчанию

---

## 5. ТЕСТОВЫЕ СЦЕНАРИИ (для QA)

1. Eyedropper на известном пикселе: синтетическое изображение 10x10, пиксель (5,5) = красный → возвращает `#FF0000`
2. Dominant colors на изображении из 2 цветов (50/50 красный/синий): должен вернуть 2 кластера ~50% каждый
3. Suggest-mappings: красно-синий дизайн + target `["#00FF00", "#FFFF00"]` → красный→зелёный, синий→жёлтый (или наоборот, по Delta-E)
4. Полностью прозрачное изображение → пустые результаты
5. Координаты за пределами → ValueError
